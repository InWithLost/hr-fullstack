# Структура проекта HR-агентства

## 1. backend — Laravel (API, бизнес-логика)
- **routes/** — маршруты API (api.php, web.php)
- **app/** — основная логика Laravel-приложения (модели, контроллеры, сервисы)
- **config/** — конфигурация Laravel
- **database/** — миграции, сиды, фабрики
- **public/** — публичные файлы (index.php, ассеты)
- **resources/** — blade-шаблоны, локализация, email-шаблоны
- **storage/** — логи, кэш, файлы загрузок
- **tests/** — тесты Laravel
- **.env** — настройки окружения (БД, почта и др.)
- **composer.json** — зависимости PHP/Laravel
- **package.json** — фронтенд-зависимости (если есть)

## 2. frontend — Nuxt 3 (SPA, клиентский интерфейс)
- **app.vue** — корневой компонент приложения
- **nuxt.config.ts** — конфигурация Nuxt (подключение Tailwind, модулей)
- **pages/** — страницы приложения (index.vue — лендинг, др. страницы)
- **components/** — Vue-компоненты (AppHeader.vue, AppFooter.vue, FeaturedJobs.vue и др.)
- **assets/css/tailwind.css** — основной CSS-файл с директивами Tailwind
- **tailwind.config.js** — конфиг Tailwind (content, theme, plugins)
- **postcss.config.js** — конфиг PostCSS (Tailwind, Autoprefixer)
- **public/** — статические ассеты (лого, изображения)
- **package.json** — зависимости Nuxt, Tailwind, PostCSS

### Особенности Nuxt 3 frontend:
- Используется официальный модуль `@nuxtjs/tailwindcss` для гарантированной интеграции Tailwind с Nuxt 3.
- Современная адаптивная верстка с использованием utility-классов Tailwind.
- Главная страница — лендинг с секциями: hero, преимущества, featured jobs, как мы работаем, FAQ, новости.
- Компоненты вынесены в отдельные файлы для масштабирования и переиспользования.
- Навигация реализована через NuxtLink (SPA-роутинг).
- **Notification store:** реализован глобальный стор уведомлений на Pinia (`frontend/stores/notification.ts`). Позволяет показывать всплывающие сообщения (успех, ошибка, информация) из любого компонента. Пример использования: `notification.show('Текст', 'success')`. Это обеспечивает единый UX для всех уведомлений во frontend.

### Важно:
- Фронтенд полностью отделён от backend, обращается к API через HTTP.
- Все стили хранятся и собираются только во frontend (Tailwind).
- Для запуска frontend используйте:
  - `npm install`
  - `npm run dev`
- Для запуска backend используйте:
  - `composer install`
  - `php artisan migrate`
  - `php artisan serve`

NOTE: На время разработки фронта middleware admin.js во фронте временно отключен (в файле Front-new/middleware/admin.js добавлен return в начале). Для защиты админки не забудьте вернуть проверку авторизации перед релизом!

# Проект готов к развитию

Решение ошибки SQLite:
- Ошибка 'could not find driver' означает, что драйвер SQLite не установлен или не активирован в PHP.
- Для использования SQLite: откройте php.ini и раскомментируйте строки extension=pdo_sqlite и extension=sqlite3, затем перезапустите PHP-сервер.
- Если вы предпочитаете использовать другую БД (например, MySQL), обновите настройки подключения в файле .env, установив DB_CONNECTION=mysql и указав соответствующие параметры, либо измените SESSION_DRIVER на file для хранения сессий в файловой системе.

# Хронология настройки окружения (май 2025)

1. Установка XAMPP и MariaDB (MySQL):
   - Были выявлены проблемы с запуском MariaDB: отсутствовала папка lib/plugin.
   - Принято решение полностью переустановить XAMPP.
   - После переустановки MariaDB успешно запускается через XAMPP Control Panel.

2. Проверка портов:
   - Порт 3306 был занят другим процессом mysqld.exe.
   - С помощью netstat и диспетчера задач лишний процесс был завершён.
   - После этого MariaDB успешно стартовала на порту 3306.

3. Создание базы данных:
   - В phpMyAdmin создана база данных hr_agency_db с кодировкой utf8mb4_general_ci.

4. Настройка Laravel (.env):
   - В файле .env указаны параметры подключения к базе данных:
     DB_CONNECTION=mysql
     DB_HOST=127.0.0.1
     DB_PORT=3306
     DB_DATABASE=hr_agency_db
     DB_USERNAME=root
     DB_PASSWORD=
   - Все параметры раскомментированы, кроме DB_PASSWORD (пароль пустой по умолчанию).

5. Миграции Laravel:
   - Команда php artisan migrate успешно выполнена из папки backend.
   - Созданы таблицы users, cache, jobs, migrations и др. в базе hr_agency_db.

6. Проверка работы:
   - Запущен сервер Laravel командой php artisan serve.
   - В браузере по адресу http://127.0.0.1:8000 отображается стартовая страница Laravel без ошибок.

7. Итог:
   - Окружение полностью готово для дальнейшей разработки.
   - Все действия и изменения зафиксированы в этом файле.
