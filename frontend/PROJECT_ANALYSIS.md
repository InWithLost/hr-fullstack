# Анализ фронтенд-проекта HR WindSurf

## Общая логика проекта

HR WindSurf - это платформа для поиска работы и подбора персонала со следующими ключевыми функциональностями:

1. **Для соискателей**:
   - Регистрация и авторизация
   - Заполнение профиля и анкеты
   - Просмотр и отклик на вакансии
   - Отслеживание статуса откликов
   - Управление образованием и опытом работы

2. **Для компаний**:
   - Регистрация и управление профилем компании
   - Создание и редактирование вакансий
   - Управление балансом и платными услугами
   - Заказ услуги подбора кандидатов

3. **Для администраторов**:
   - Управление пользователями
   - Управление компаниями
   - Модерация вакансий
   - Управление контентом блога
   - Управление специальностями

## Структура проекта

Проект построен на Nuxt.js с использованием Vue 3 и следует стандартной структуре Nuxt-проекта:

- `/pages` - страницы приложения с файловой маршрутизацией
- `/components` - многоразовые компоненты
- `/layouts` - макеты страниц
- `/middleware` - промежуточное ПО для проверки авторизации
- `/stores` - хранилища состояния (Pinia)
- `/api` - схемы и интерфейсы для работы с API
- `/utils` - вспомогательные функции
- `/public` - статические файлы
- `/assets` - ресурсы (изображения, стили)

## Ключевые компоненты и их назначение

### Компоненты интерфейса
- `AppHeader.vue` - главный заголовок сайта с навигацией
- `AppFooter.vue` - подвал сайта
- `AdminHeader.vue` - заголовок для административной панели
- `CompanyProfileHeader.vue` - заголовок для профиля компании
- `ProfileNavigation.vue` - навигация в профиле пользователя
- `Notification.vue` и `NotificationContainer.vue` - система уведомлений

### Компоненты для работы с вакансиями
- `JobCard.vue` - карточка вакансии для списков
- `JobEditForm.vue` - форма создания/редактирования вакансии для компаний
- `JobEditFormAdmin.vue` - форма создания/редактирования вакансии для администраторов
- `FeaturedJobs.vue` - блок с избранными вакансиями на главной странице

### Компоненты для работы с контентом
- `BlogCard.vue` - карточка статьи блога
- `SpecialtyCard.vue` - карточка специальности
- `RichTextEditor.vue` - редактор форматированного текста (TinyMCE)
- `SimpleRichTextEditor.vue` - упрощенный редактор текста

### Вспомогательные компоненты
- `ImageUploader.vue` - компонент для загрузки изображений
- `ConfirmDialog.vue` - диалог подтверждения действий
- `RecruitmentForm.vue` - форма заявки на подбор персонала

## Основные страницы и их функциональность

### Публичные страницы
- `/` (index.vue) - главная страница с поиском вакансий и формой заявки
- `/jobs` - список всех вакансий с фильтрацией
- `/jobs/[slug]` - детальная страница вакансии
- `/specialties` - список специальностей
- `/specialties/[slug]` - вакансии по специальности
- `/blog` - блог с полезными статьями
- `/blog/[slug]` - отдельная статья блога
- `/contacts` - контактная информация
- `/login` и `/register` - авторизация и регистрация
- `/legal/*` - юридические документы

### Профиль пользователя
- `/profile` - основная информация профиля
- `/profile/anketa` - анкета соискателя
- `/profile/education` - образование и опыт
- `/profile/applications` - отклики на вакансии

### Профиль компании
- `/company/register` - регистрация компании
- `/company/profile` - информация о компании
- `/company/profile/edit` - редактирование профиля
- `/company/profile/jobs` - управление вакансиями
- `/company/profile/jobs/create` - создание вакансии
- `/company/profile/jobs/[id]/edit` - редактирование вакансии
- `/company/profile/balance` - управление балансом

### Административная панель
- `/admin` - главная страница админки
- `/admin/users` - управление пользователями
- `/admin/companies` - управление компаниями
- `/admin/jobs` - управление вакансиями
- `/admin/blog` - управление блогом
- `/admin/specialties` - управление специальностями
- `/admin/applications` - просмотр откликов на вакансии

## Особенности реализации

1. **Авторизация**:
   - Используется JWT-токен, хранящийся в localStorage
   - Middleware проверяет наличие токена и перенаправляет неавторизованных пользователей

2. **Управление состоянием**:
   - Используется Pinia для глобального состояния
   - Реактивные переменные Vue для локального состояния компонентов

3. **Работа с API**:
   - В демо-режиме используются моковые данные
   - Предусмотрены комментарии для реальных API-запросов

4. **Роли пользователей**:
   - Соискатель
   - Компания (управляющий компанией)
   - Администратор

5. **Баланс компании**:
   - Денежный баланс для платных услуг
   - Бесплатные размещения вакансий
   - История транзакций

## Неиспользуемые или избыточные файлы

1. **Дублирующиеся файлы**:
   - `AppHeader.temp.vue` - временная версия заголовка
   - `admin/jobs/jobs.vue` - дублирует функциональность `admin/jobs/index.vue`
   - `balance.vue.new` - вероятно, новая версия страницы баланса

2. **Устаревшие файлы**:
   - `admin/job-create.vue` и `admin/job-edit.vue` - заменены на `/admin/jobs/create.vue` и `/admin/jobs/[id]/edit.vue`
   - `admin/view-application.vue` - заменен на `/admin/applications/[id]/index.vue`

3. **Файлы с неясным назначением**:
   - `fix-duplicate.js` - вспомогательный скрипт, назначение неясно
   - `scripts/create-company-user.js` - вероятно, скрипт для создания тестовых данных
